# Design To Frontend Quality

## User Goal

Use design-craft-gate, DesignDNA, anti-slop, and UI framework skills to build a polished product surface with product/design context, accessibility, responsive layout, deterministic anti-pattern checks, and implementation evidence.

## Expected Workflow

PRODUCT/DESIGN context gate -> shape brief -> brand/design system selection -> UI framework adapter -> frontend patterns -> deterministic design-audit -> visual/accessibility QA -> output quality gate.

## Acceptance Evidence

- design-craft-gate is available and requires PRODUCT.md / DESIGN.md context or explicit assumptions before UI mutation.
- DesignDNA is available with brand systems and generated artifacts.
- Frontend/design skills are discoverable without duplicate installable names.
- DesignDNA CLI tests, deterministic anti-pattern audit, and brand synchronization checks pass.
