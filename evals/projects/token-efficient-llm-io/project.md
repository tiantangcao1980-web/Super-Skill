# Token-Efficient LLM Input And Output

## User Goal

Improve LLM input/output quality while saving tokens and preserving larger context through contracts, cache layering, structured outputs, and quality gates.

## Expected Workflow

Intent contract -> context engineering -> token budgeting -> prompt cache layering -> model adaptation contract -> output quality gate -> verification report.

## Acceptance Evidence

- Always-on context is separated from volatile evidence.
- Model routing and output schemas are explicit.
- Final output must include changed files, evidence, risks, and known gaps.
