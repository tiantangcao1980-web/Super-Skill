# Ultra-Lite Engineering Discipline

This fixture validates that Super Skill can compress lessons from
Superpowers, Karpathy-style guardrails, and Matt Pocock-style project context
into a small installable profile instead of forcing every agent to load the
entire lifecycle collection.

Expected workflow:

1. Use `engineering-core-loop` as the always-on compact loop: route, contract,
   TDD or named exception, small implementation, review, verification, finish,
   and memory hygiene.
2. Use `goal-driven-workflow` and `intent-contract` to express Done when,
   Stop if, scope, constraints, and token budget before implementation.
3. Use `karpathy-discipline` to keep the change simple, surgical, and
   assumption-aware.
4. Use `test-driven-development`, `code-review`, `output-quality-gate`, and
   `verification-loop` so every completion claim has evidence.
5. Use `domain-context-adr` to create CONTEXT.md and ADR records only when
   shared language or hard-to-reverse architecture decisions need durability.
6. Use `safe-command-governance` before documenting or running risky commands,
   including curl-pipe-shell, rm -rf, git reset --hard, chmod 777, or forceful
   overwrite examples.
7. Send only verified, deduplicated lessons to `agent-memory-dream-loop`; do
   not store raw prompts, raw responses, or private transcripts as memory.

Required phrases: Superpowers, Karpathy, CONTEXT.md, ADR, TDD, safe command,
Done when, Stop if, verification, memory.
