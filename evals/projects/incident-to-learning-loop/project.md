# Incident To Learning Loop

## User Goal

Turn a production incident into diagnosis, fix validation, negative memory, and skill or eval improvement without unsafe automatic mutation.

## Expected Workflow

Observability triage -> debugging -> security/quality review -> regression eval -> memory candidate -> dream replay -> skill lifecycle gate.

## Acceptance Evidence

- Observability, debugging, review, eval, memory, and skill evolution skills exist.
- Memory promotion requires evidence, redaction, expiry, and verification.
- Skill curation protects core skills, requires dedup before create, and archives reversibly.
