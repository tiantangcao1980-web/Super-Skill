# Apple — BANNED Patterns

## Typography
- NEVER use Inter, Roboto, Open Sans — Apple uses SF Pro exclusively
- NEVER use `font-weight: 700` or higher in body or UI copy — Apple stays at 300-600
- NEVER use ALL-CAPS headings larger than 12px eyebrows
- NEVER mix serif with SF Pro — one family rules the page
- NEVER use a monospace font outside of code blocks

## Color
- NEVER use pure `#000000` — use `#1d1d1f` warm near-black
- NEVER use saturated accent colors (no neons, no hot pink, no emerald-on-white)
- NEVER use more than ONE chromatic accent per page (Apple Blue `#0071e3`)
- NEVER use gradient text or gradient fills except on product photography
- NEVER use colored shadows (drop shadows are always neutral black with low alpha)

## Layout
- NEVER use 3-column feature card grids
- NEVER center everything — Apple uses generous left-aligned text on product pages
- NEVER use tight containers; Apple's text has `max-width: 560px` and sits inside 100vw frames
- NEVER decorate backgrounds (no mesh gradients, no noise, no grid overlays)
- NEVER break the product-photo-first hierarchy — imagery dominates, text supports

## Components
- NEVER use heavy borders; Apple prefers zero-border with padding + background color
- NEVER use pill-shaped buttons; Apple uses `border-radius: 980px` for pills but only for discovery CTAs, otherwise 8-12px
- NEVER use cards with glow / gradient / frosted glass
- NEVER use icon sets from third parties — SF Symbols or nothing

## Motion
- NEVER use bouncy spring animations — Apple uses `cubic-bezier(0.25, 0.1, 0.25, 1)`
- NEVER exceed 400ms for interaction feedback
- NEVER auto-play video without mute + loop

## Copy
- NEVER use "effortlessly," "seamlessly," "revolutionary" in Apple's voice
- NEVER use emoji in headings
- NEVER use fake metrics — Apple states specs, not marketing numbers
- Headlines are declarative nouns, not verb commands ("Ultra performance." not "Experience ultra performance")

## Signature "don'ts"
- NEVER add decorative borders around product shots — the photo IS the frame
- NEVER stack multiple CTAs; Apple uses ONE primary action, optionally one secondary link
- NEVER use testimonials — Apple shows the product, not opinions about the product
