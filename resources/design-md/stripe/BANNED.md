# Stripe — BANNED Patterns

## Typography
- NEVER use heavy weights (700+) on headings — Stripe headlines use **weight 300-400**
- NEVER use system default sans; Stripe uses `sohne-var` with OpenType `ss01`
- NEVER use tight letter-spacing on display type — Stripe uses `-0.01em` to `0.01em`, subtle
- NEVER use italics except for inline product names

## Color
- NEVER use pure black `#000` — Stripe uses `#0a2540` or `#061b31` deep navy
- NEVER use saturated hot pink or red-orange as accent — Stripe's palette is cool purples + navy
- NEVER use yellow or orange accents (except payment-method icons)
- NEVER use high-contrast borders — Stripe borders are `#e5edf5` subtle

## Layout
- NEVER use dense bento grids — Stripe breathes with 80-120px vertical gaps
- NEVER use hard dividers between sections — whitespace is the separator
- NEVER show more than 3 product teases on one page
- NEVER use full-bleed imagery except for hero (content is constrained to `max-w-7xl`)

## Components
- NEVER use sharp corners on cards — Stripe uses `border-radius: 12px` minimum
- NEVER use drop shadows with hard edges — Stripe uses `0 20px 50px -10px rgba(5,5,93,0.1)` blue-tinted
- NEVER use emoji or illustration for feature icons — Stripe uses branded line icons
- NEVER use sticky navigation that inverts color on scroll

## Motion
- NEVER use spring animations with overshoot — Stripe uses `ease-out` curves
- NEVER use hover animations > 300ms
- NEVER animate the Stripe gradient mark — it's a static vector

## Copy
- NEVER use "disrupting," "revolutionary," or hype language — Stripe's voice is technical-plain
- NEVER use exclamation marks
- NEVER use second-person "You" as the opening word of a headline
- Headlines start with the capability or object ("Payments for platforms")

## Signature "don'ts"
- NEVER use Stripe's gradient as a color fill for text — it's a product mark only
- NEVER place the logo anywhere but top-left of the nav
- NEVER style pricing with strikethrough "was $X now $Y" discounts — Stripe shows actual prices
