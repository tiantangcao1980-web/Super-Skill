# Research To Delivery Workflow

Use this flow when a project starts from an idea, rough requirement, product bet, feature request, or redesign request.

## 0. Intake

Skills:

- `intent-contract`
- `context-engineering`
- `token-budgeting`

Clarify:

- User, buyer, operator, and stakeholder
- Core problem and current workaround
- Business goal and risk
- Constraints: budget, time, stack, data, compliance, launch channel

Outputs:

- Intent contract
- Context pack
- Token budget and ignored-source list

## 1. Research

Skills:

- `user-research`
- `market-research`
- `file-curation`

Outputs:

- Research report
- Competitor map
- Evidence gaps

## 2. Analysis

Skills:

- `requirement-analysis`
- `data-analysis`

Outputs:

- JTBD
- User stories
- Prioritization matrix
- Decision brief

## 3. Product

Skills:

- `product-spec`

Outputs:

- PRD
- MVP slice
- Success metrics
- Risks and rollback thinking

## 4. Design

Skills:

- `designdna`
- `design-templates`
- `copywriting-design`
- relevant UI library skill

Outputs:

- DESIGN.md
- Tokens
- Copy matrix
- Banned patterns

## 5. Interface

Skills:

- `api-design`
- `api-gateway`
- `cli-design`
- `create-cli`

Outputs:

- API contract
- CLI schema
- Error and exit-code model

## 6. Development

Skills:

- `frontend-patterns`
- `backend-patterns`
- `database-patterns`
- `coding-standards`
- `ai-agent-frameworks`
- `chatgpt-apps`

Outputs:

- Working implementation
- Tests around risky behavior
- Clear code ownership

## 7. Quality

Skills:

- `qa-strategy`
- `browser-automation`
- `test-generation`
- `e2e-testing`
- `debugging`
- `code-review`
- `security-review`
- `verification-loop`
- `output-quality-gate`

Outputs:

- Passing checks
- Browser or workflow evidence
- Review findings addressed
- Known risk list
- User-expectation fit check

## 8. Delivery

Skills:

- `deployment-patterns`
- `docker`
- `git`
- `github`
- `programmatic-video`

Outputs:

- Commit/PR/release
- Deployment plan
- Demo artifact
- Rollback notes

## 9. Operations

Skills:

- `documentation`
- `continuous-learning`
- `file-curation`

Outputs:

- README/docs
- Lessons learned
- Knowledge base updates
