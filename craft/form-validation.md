# 表单与校验

## P0 规则

- **错误就近显示**：错误文案紧贴对应字段，不只在 submit 顶部 banner。
- **错误描述具体**：不是 "Invalid"，而是 "邮箱缺 @ 或域名"。
- **校验时机**：失焦校验（onBlur）+ 提交时校验；不要每个 keystroke 都报错。
- **必填字段标记**：用 `*` + `aria-required="true"`，并在表单顶部说明 `*` 的含义。
- **提交按钮不要 disable 到看不见**：保持可点击，点击后再展示错误，否则用户不知道为什么不能提交。

## P1 指导

- 长表单按节切分 (`<fieldset>` + `<legend>`)，单节字段 ≤ 7。
- 自动 focus 第一个错误字段；不只滚动到顶部。
- 异步校验（用户名查重）要有 debounce + 加载态。
- 成功状态比错误状态更克制：✓ + 短反馈，不要"恭喜！"。

## P2 指导

- 密码强度提示用 progressive disclosure，不要默认展开 5 行规则。
- 验证码 OTP 字段拆成单字符 6 格输入，自动跳格、支持粘贴。
- 文件上传支持拖拽 + 预览 + 重传 + 删除。

## 反模式

- 「请输入合法的电话号码」但不告诉用户合法格式是什么。
- 提交后清空整张表单，让用户重填。
- 错误用红色边框但没有文字说明（违反 a11y "颜色不是唯一信息载体"）。
