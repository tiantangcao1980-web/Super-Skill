# DESIGN.md Template

Use this concise structure when a project needs a reusable design specification.

```markdown
# Project Design

## 1. Visual Theme
Mood, density, brand references, and constraints.

## 2. Color Roles
Semantic tokens for background, surface, text, border, accent, success, warning, danger.

## 3. Typography
Font families, scale, weights, line heights, and label treatment.

## 4. Layout
Grid, max widths, spacing scale, section rhythm, responsive rules.

## 5. Components
Buttons, inputs, cards, lists, tables, navigation, modals, empty/error states.

## 6. Motion
Timing, easing, reduced-motion behavior, interaction feedback.

## 7. Assets
Logos, icons, imagery, screenshots, generated asset provenance.

## 8. Accessibility
Contrast, focus, keyboard, touch targets, screen reader notes.

## 9. Banned Patterns
Specific visual and copy patterns that must not appear.
```
