# 持续学习

## 学习流程

### 1. 模式识别

在每次会话中关注：
- 反复出现的代码模式
- 成功的问题解决策略
- 项目特定的约定和偏好
- 常见的错误和修复方法

### 2. 知识提取

会话结束或关键节点时提取：

```markdown
## 学习记录

### 日期: [YYYY-MM-DD]
### 项目: [项目名称]

### 发现的模式
- 模式名称: [描述]
- 适用场景: [何时使用]
- 代码示例: [简短示例]

### 问题解决
- 问题: [描述]
- 根因: [分析]
- 解决方案: [步骤]
- 预防措施: [如何避免再次发生]

### 项目约定
- [记录项目特有的约定]
```

### 3. 知识分类

| 类别 | 存储位置 | 用途 |
|------|----------|------|
| 通用模式 | ~/.codex/memory/ | 跨项目复用 |
| 项目约定 | AGENTS.md | 项目内一致性 |
| 个人偏好 | ~/.codex/memory/ | 个性化配置 |

### 4. 信心评分

为每条学习记录评分：
- **高信心 (>0.8)**: 多次验证的模式
- **中信心 (0.5-0.8)**: 有效但样本不足
- **低信心 (<0.5)**: 初次观察，需进一步验证

## 自动触发

在以下情况自动提取学习：
1. 解决了一个 Bug（记录问题和修复）
2. 重构了代码（记录优化模式）
3. 发现了新的项目约定
4. 用户表达了明确偏好
